Step 7: The new setting takes effect immediately. Every user who tries to copy/ move data to USB devices or format USB drive will get the error message

Creating a USB Write Blocker Forensic Image using EnCase Imager.
I.  Creating a forensic image
II.Check integrity of data
III.Analyze forensic image
I. Creating a USB Forensic Image

Step 01: Launch EnCase and Create a New Case.
·	Open EnCase Forensic.
·	Click File > New.
·	Enter Case Information (Name, Number, Examiner) and click Finish.

Step 02: Add the Source Drive.
·	In the Left "Tree" Pane, right-click on Entries.
·	Select Add Device.
·	Locate and select the checkbox for your suspect drive (connected via a write-blocker).
·	Click OK. The drive will appear in the Tree Pane.

Step 03: Start the Acquisition.
·	Right-click on the suspect drive in the Tree Pane.
·	Select Acquire... from the menu.

Step 04: Configure Image Settings.
·	Destination: Browse to your destination folder.
·	File Name: Enter a name for the image file (e.g., MyEvidence).
·	Format: Select Expert Witness Format (Ex01).
·	Segment Size: Set a size (e.g., 2 GB).
·	Compression: Select Fast.
·	Checksum: Ensure MD5 and SHA-1 are selected.
·	Click Start.

Step 05: Imaging Process.
·	EnCase will begin acquiring the image.
·	A progress window will show the status, time elapsed, and the calculated hash values.

Step 06: Completion.
·	When complete, a message will show "Acquire is complete."
·	The MD5 and SHA-1 hashes are displayed and automatically verified against the source.

II. Check Integrity of Data
·	The integrity check is automatic in EnCase.
·	After acquisition, EnCase compares the hash (MD5/SHA-1) of the source device with the hash of the created image file.
·	A confirmation message verifies the hashes match, proving the image is a perfect, forensically sound copy.

III. Analyze USB Forensic Image
Step 01: Add the Image to a Case.
·	Go to File > Add Evidence Item....
·	Select Image File and click Next.
·	Browse to and select your .Ex01 image file. – Click finish.

Step 02: Explore the Evidence.
·	The image file is now loaded in the Tree Pane.
·	You can browse the file system, view files, recover deleted data, and analyze the evidence.
Done!

2. Display packets which are coming from specific IP-address
Filter: ip.src == 192.168.10.28
 
3. Display packets which are having specific IP-address destination
Filter: ip.dst == 192.168.10.28

 
4. Display packets which are using http protocol
Filter: http 
 
5. Display packets which are using http request
Filter: http.request
 
6. Display packets which are using TCP protocol
Filter: tcp 
 
7. Display packets having no error connecting to server
Filter: http.response.code==200
 
8. Display packets having port number 80, 443
Filter: tcp.port==80 || udp.port==443
 
9. Display packets which that contains keyword facebook
Filter: tcp contains facebook
